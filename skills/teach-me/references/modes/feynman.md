# teach-me mode — feynman (Feynman technique)

## Usage

```text
/teach-me --feynman <topic>    Claude Code / Cursor / Antigravity
/teach-me --fey <topic>        short alias
$teach-me --feynman <topic>    Codex / OpenCode
/teach-me --feynman            debrief: explain the work you just shipped
```

Mid-session: say "switch to feynman" (any language works, e.g. "换费曼式") — takes effect
immediately, no re-invocation needed.

## When to choose / when NOT

Choose to consolidate material the learner has already met: work just done with an agent
(pairs especially well with the no-topic debrief), a topic read but wobbly, interview
prep. Do NOT choose for first contact with a topic — with zero exposure there is nothing
to explain yet; let the shared base loop teach it first, then use feynman to consolidate.

## Turn protocol — the four-step cycle (per concept)

1. **Learner explains first.** Ask the learner to explain the concept to an imagined
   novice, in plain language. The coach's opening is one sentence plus the request; the
   learner talks first from turn one.
2. **Locate the gap together.** Point at ONE spot: a skipped step, a fuzzy cause, or
   jargon-hiding (a term used in place of an explanation). Name it specifically. The
   coach may ban a term: "explain it without saying 'cache'."
3. **Fill only that gap.** The coach explains the located gap minimally — one to three
   sentences or one small example. Nothing else.
4. **Re-explain, simpler.** The learner explains again, simpler than before, adding one
   analogy. Give the analogy one boundary check: "where does it break down?"

Repeat 2–4 until the explanation stands, then move to application.

## Fluency guard (the failure mode this mode exists to block)

A fluent explanation is E1 evidence ONLY. `demonstrated` still requires E2 (apply unaided)
and E3 (transfer). After the cycle, always set one application task; never let a smooth
step-4 explanation close the concept.

## Overrides (vs the shared base loop)

- Who talks first: the learner, from the opening turn (the base loop opens with activation
  plus a minimal explanation).
- The coach's telling is restricted to step 3 gap-filling and stays minimal.

## Never overrides

The E1/E2/E3 rubric, state machine, fatigue hard rules, scaffolding ladder,
one-main-action rule, source discipline, and the write-consent gates in `teaching.md` /
`state.md` apply unchanged.

## Red flags

- Marking `demonstrated` from a fluent explanation without E2/E3 → STOP, fluency guard.
- The coach lecturing outside a located gap → STOP, that is the base loop's job, not feynman's.
- Letting jargon pass as explanation → STOP, ban the term and ask for plain words.
- Demanding a full re-explanation after a stall instead of laddering the stuck step → STOP.
